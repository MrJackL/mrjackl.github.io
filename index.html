CocoaPods使用技巧

Sun, Jan 24, 2016

安装和基本使用CocoaPods是十分简单的事情，但是在日常项目当中难免会出现一些奇葩状况，以下列出可能遇到的问题，以及解决办法。

1. 执行pod install 或者 pod update 一直卡住不动，并提示Updating local specs repositories ; (也许是被墙了)这个时候需要执行以下脚本:

pod install --verbose --no-repo-update

或者

pod update --verbose --no-repo-update

2. 使用pod search 或者pod update 时，查询出的第三方库版本低于实际版本； 逐个执行以下脚本:

brew install ruby

sudo gem install cocoapods

pod setup

pod repo update --verbose

CocoaPods使用教程

Sun, Jan 24, 2016

在Github上有很多开源的iOS项目，大部分都支持pods,那么我们该如何将大神们的开源库，通过CocoaPods集成到我们自己的项目中呢？

1. 生成Podfile

首先创建一个iOS项目Demo, 打开Terminal，并输入 cd , 然后将项目文件夹拖入到Terminal，然后Enter；这样就进入到了项目的路径下，

然后输入以下脚本：

touch Podfile

此时在你的项目文件夹中就会生成一个Podfile文件 ;

2.编辑Podfile

双击打开Podfile, 将以下内容保存到Podfile中:

platform :ios, '7.0'

pod "AFNetworking"

3. 执行install

在Terminal 中cd 项目路径,执行以下脚本:

pod install --verbose --no-repo-update

或者

pod update --verbose --no-repo-update

耐心等待几秒，脚本执行完成后，去项目路径下查看，会多出3个文件，如下图:

如果执行结果和这个一样，就代表第三方库已经引用成功了！另外，这里需要注意是的，以后启动项目就要双击xcworkspace文件了。

这里只是以AFNetworking为例，当然如果你需要在项目中引用其他第三方库，只需要编辑Podfile，将你需要的第三方库pods加入到Podfile文件中，然后执行install或者update 。

CocoaPods安装

Sun, Jan 24, 2016

  • 介绍

    CocoaPods是iOS中使用最受欢迎的第三方库管理工具，使iOS开发者的对项目的管理变得更加简单；现在有了CocoaPods,就可以通过一段更新脚本就完成了所有第三方类库的更新；而且在Github上有很多开源的项目，大多都支持pods。

    • 前提:Ruby环境

    需要先安装好Ruby环境，这里就不详细介绍了，链接说的很清楚。https://ruby-china.org/wiki/install_ruby_guide

    • 安装CocoaPods

    安装好Ruby环境后，打开Terminal(终端)，理论上只要运行以下脚本就可以安装好CocoaPods:

 sudo gem install cocoapods

但是cocoapods.org被墙了，只好用淘宝的Ruby镜像来访问cocoapods 。

 gem sources --remove https://rubygems.org/

以上脚本执行完成后，再执行以下脚本:

gem sources -a http://ruby.taobao.org/

可以通过以下脚本验证你刚才使用的Ruby镜像是taobao:

gem sources -l

这个时候，你只需要在Terminal中运行：

$ sudo gem install cocoapods

坐等一会，CocoaPods就在你本地下载并且安装好了，大功告成！
