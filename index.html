iOS入门-前言

Thu, Jan 28, 2016


注：本文章适合iOS初学者查看,本文将以Objective-C为例进行讲解.

很多刚接触iOS开发的同学，总是会疑惑到底该如何去学习iOS，该从哪里下手，熟悉哪些方面才算入门了？

iOS App的开发主要是涉及到UI布局、数据处理、使用第三方库、业务逻辑 。(后面会有对应的文章进行讲解) 对于初学者来说，最快理解iOS的方式就是UI布局，在这个过程中不但可以熟悉iOS的语法、控件，而且能带来最明显的成就感。个人建议初学者，可以多创建一些Demo来练手；

接下来会从iOS开发工具、创建项目、项目结构、程序启动流程进行介绍:

1.Xcode

Xcode是iOS开发工具，有自带模拟器，可以方便的进行调试，有各种分辨率的模拟设备;

现在来安装Xcode, 在Mac App Store中搜索”Xcode”，安装即可；（不过下载速度确实会有点慢）

2.创建项目

1.打开Xcode,用快捷键Command+Shift+N 就会出现创建工程的视图框，一般选择Page-Base Application ;如图:

2.填写项目名称 (这里取名Demo);同时这里需要Language(Objective-c 或者 Swift),Device(iPhone,iPad,Universal) ,初学者可以先选择（Objective-C ,iPhone）即可。如图:

3.最后一步是选择项目文件保存的位置，建议直接保存在桌面；走完这3步之后，你的第一个iOS项目就正式创建成功了！

3.项目结构

创建完成之后，项目结构如下图:

你在桌面也可以看到Demo文件夹，以及内部的文件。

AppDelegate 程序启动执行文件 ;
Assets.xcassets 要适配不同分辨率的设备，必须提供不同分辨率的启动图片，以及AppIcon
LaunchScreen.storyboard 启动视图（俗称:闪屏）
Main.storyboard 程序视图文件
Info.plist 程序配置文件（配置程序启动时所使用的storyboad文件，还有很多配置信息以后要用到再讲）
.h .m文件 .h文件是指头文件，作用:将方法暴露给外部调用 ;.m文件是指实现文件，作用:内部逻辑，方法实现等等.

注:如果这里不清楚storyboard，可以查看Apple官网说明 Storyboard


4.程序启动流程

Xcode运行程序的快捷键:Command+R ;程序启动时，首先会调用AppDelegate文件中的didFinishLaunchingWithOptions方法;也许你会觉得奇怪，这个方法里面什么内容都没有为什么模拟器里面会有内容呢？

刚才在项目结构里面有提到Info.plist文件，其中Main storyboard file base name就是绑定了Main.storyboard,这就意味着程序启动的时候会自动调用这个文件；


小结

初学者只需按照以上步骤手动操作即可，不必太过刻意去理解每个文件的含义。很多路都是靠走出来的，只有不停的动手操作才会碰到问题，然后去解决问题，最终就有了思考，有了理论。

CocoaPods使用技巧

Sun, Jan 24, 2016

安装和基本使用CocoaPods是十分简单的事情，但是在日常项目当中难免会出现一些奇葩状况，以下列出可能遇到的问题，以及解决办法。

1. 执行pod install 或者 pod update 一直卡住不动，并提示Updating local specs repositories ; (也许是被墙了)这个时候需要执行以下脚本:

pod install --verbose --no-repo-update

或者

pod update --verbose --no-repo-update

2. 使用pod search 或者pod update 时，查询出的第三方库版本低于实际版本； 逐个执行以下脚本:

brew install ruby

sudo gem install cocoapods

pod setup

pod repo update --verbose

CocoaPods使用教程

Sun, Jan 24, 2016

在Github上有很多开源的iOS项目，大部分都支持pods,那么我们该如何将大神们的开源库，通过CocoaPods集成到我们自己的项目中呢？

1. 生成Podfile

首先创建一个iOS项目Demo, 打开Terminal，并输入 cd , 然后将项目文件夹拖入到Terminal，然后Enter；这样就进入到了项目的路径下，

然后输入以下脚本：

touch Podfile

此时在你的项目文件夹中就会生成一个Podfile文件 ;

2.编辑Podfile

双击打开Podfile, 将以下内容保存到Podfile中:

platform :ios, '7.0'

pod "AFNetworking"

3. 执行install

在Terminal 中cd 项目路径,执行以下脚本:

pod install --verbose --no-repo-update

或者

pod update --verbose --no-repo-update

耐心等待几秒，脚本执行完成后，去项目路径下查看，会多出3个文件，如下图:

如果执行结果和这个一样，就代表第三方库已经引用成功了！另外，这里需要注意是的，以后启动项目就要双击xcworkspace文件了。

这里只是以AFNetworking为例，当然如果你需要在项目中引用其他第三方库，只需要编辑Podfile，将你需要的第三方库pods加入到Podfile文件中，然后执行install或者update 。

CocoaPods安装

Sun, Jan 24, 2016

  • 介绍

    CocoaPods是iOS中使用最受欢迎的第三方库管理工具，使iOS开发者的对项目的管理变得更加简单；现在有了CocoaPods,就可以通过一段更新脚本就完成了所有第三方类库的更新；而且在Github上有很多开源的项目，大多都支持pods。

    • 前提:Ruby环境

    需要先安装好Ruby环境，这里就不详细介绍了，链接说的很清楚。https://ruby-china.org/wiki/install_ruby_guide

    • 安装CocoaPods

    安装好Ruby环境后，打开Terminal(终端)，理论上只要运行以下脚本就可以安装好CocoaPods:

 sudo gem install cocoapods

但是cocoapods.org被墙了，只好用淘宝的Ruby镜像来访问cocoapods 。

 gem sources --remove https://rubygems.org/

以上脚本执行完成后，再执行以下脚本:

gem sources -a http://ruby.taobao.org/

可以通过以下脚本验证你刚才使用的Ruby镜像是taobao:

gem sources -l

这个时候，你只需要在Terminal中运行：

$ sudo gem install cocoapods

坐等一会，CocoaPods就在你本地下载并且安装好了，大功告成！
